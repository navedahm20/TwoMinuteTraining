ID Practices That Need To Evolve

There are a couple of practices that we're involved in that the presenters argued, must evolve. These two practices are: how we use metrics and how we analyze problems.

How We Use Metrics

When we think about using metrics, what do we usually think of? Typically, we develop a learning solution, then we check to see if team member performance improved, and then we check to see if it impacted the business (that is, if we even get to that point).

The presenters argued that we need to actually do this in reverse order. We should begin by analyzing business metrics. These are things like costs, revenue, market share, volume, and speed. Then, we can look at performance metrics. These are productivity measures and, excuse my french, key performance indicators (remember those?). Then, we can start to develop our learning solution metrics. These are defined as indicators that the learning solution had some kind of impact on the performance metrics.

What exactly is meant by "indicators of impact" for a learning solution. They would be the answers to the following questions:

  • Are the right people using the solution in the right context?
  • Is the solution readily available when it's needed?
  • Is it easy to use?
  • Is it relevant and helpful?
  • Are enough people using it?
  • How is it affecting team/business metrics?
